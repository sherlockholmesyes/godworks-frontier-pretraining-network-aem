# AEM Missing Bricks

## design root

Rejected option A:

```text
AEM needs a distributed trainer.
```

Death: a trainer alone copies the corporate form: one run, one checkpoint, one training fabric.

Rejected option B:

```text
AEM needs a marketplace of LoRA/expert files.
```

Death: a marketplace without admission, routing, verification, and distillation becomes an adapter landfill.

Third:

```text
AEM needs a protocol stack for verified capability growth.
```

## Missing bricks

```text
1. Expert Capsule ABI
2. TrainingReceipt format
3. Task-level Router
4. Objective VerifierReport gate
5. DataShard Ledger
6. Sandbox verifier for code experts
7. 12GB QLoRA training receipt path
8. RouteTrace / Distillation Memory
9. Node daemon with signatures
10. Reputation/Credit layer
11. TeacherPolicyCard for legal distillation
12. DistillationReceipt for audited teacher-signal use
```

## First kill gate

```text
Can AEM reject a fake expert with beautiful manifest claims unless an objective verifier report passes?
```

fake expert claims + real verifier rejects it
